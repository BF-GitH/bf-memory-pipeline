Reflection / Consolidation
Every N turns, runs a maintenance pass that DEDUPES the fact database (merging near-duplicate facts) and makes ONE extra LLM call (reusing Agent 3's profile) to write higher-order observation facts. Conservative + cost-aware — it runs infrequently, off the critical path, and never blocks the reply.
Successful pipeline runs between reflection passes. Higher = cheaper, less frequent maintenance.
